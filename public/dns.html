Instalación y configuración del Servidor DNS

💥 Paso 1

Ahora se procede a agregar un nuevo rol seleccionando en la opción Servidor DNS que nos permitirá hacer una traducción de nombres de dominio a direcciones IPv4 en este caso y viceversa

💥 Paso 2

Siguiendos todos los pasos para la instalación y configuración de este rol tendremos instalado nuestro rol para el servidor DNS, en el cual se procede a configurar.

💥 Paso 3

Windows Server nos permite para el servidor DNS asignar distintas zonas en las cuales va a tener información de los dominios que se van a traducir. Por lo tanto se crea una zona directa la cual va a tomar la dirección IP que se le asigne y va a trabajar sobre ella para traducir los nombres de dominio.

💥 Paso 4

Vamos a crear una zona principal puesto que estamos trabajando directamente en el servidor que ofrece el servicio

💥 Paso 5

Ahora asignamos el nombre de la zona con el fin de identificarla. Cabe resaltar que este nombre no es el nombre propio del servidor DNS

💥 Paso 6

Windows Server permite trabajar sobre IPv4 o IPv6. Para nuestro caso usaremos direcciones IPv4

💥 Paso 7

En esta parte asignaremos la dirección IP que trabaja sobre esa zona. En este caso será la misma dirección de red que trabaja el host. 192.168.100.X. Esta dirección se identificará comunmente como el inverso de la dirección ip de red, como se aprecia en la zona inferior de la imagen, creando así una zona de búsqueda inversa.

💥 Paso 8

Aquí se aprecia que fue creada la zona de búsqueda directa con el nombre asignado

💥 Paso 9

Automáticamente se crea la zona de búsqueda inversa con la dirección IP de la parte de red que fue asignada.

💥 Paso 10

Ahora vamos a crear los host que son las direcciones IP que son traducidas hacia nuestro nombre de dominio, por lo que si en el navegador digitamos el nombre de dominio, el servidor DNS nos traducirá ese dominio a una dirección IP a la que se conectará nuestro cliente

💥 Paso 11

Aquí podemos ver los distintos host que se han creado. En este caso, para la IP 192.168.100.102. También dispondremos hacia el nombre de dominio el acceso por www para simular un nombre de dominio típico de internet

💥 Paso 12

Por último, se realiza el test directamente en la consola de comandos del servidor, para verificar la resolución de los nombres de dominio e IP's. El comando nslookup, tanto para la dirección IP como para el nombre de dominio. De esta manera, estaría configurado, instalado y montado nuestro servidor DNS.
